Llave Primaria:
    Las tablas deben tener un campo o conjunto de campos que identifique de forma unica un registro, a este campo o conjunto de campos se le llama LLAVE PRIMARIA
    La clave subrogada o surrogate key es el identificador único de una tabla que no se deriva de los datos de la aplicación, generalmente no es visible al usuario y se suele construir a partir de una secuencia autogenerada.

Llave Foranea:
    Es un campo o conjunto campos que provienen de otra tabla para establecer una relacion.


Pasos o guía para crear un modelo relacional:

1. Listar tablas o entidades
2. Listar atributos de las tablas
3. Crear relaciones basicas de las tablas encontradas
4. Identificar atributos que deberían ser nuevas tablas o entidades (Normalizar)
5. Repetir en caso de ser necesario.
